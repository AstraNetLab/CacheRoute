
LMCache is an open-source caching system designed to improve the efficiency of large language model (LLM) inference by reusing previously computed key-value (KV) cache data. It targets one of the major inefficiencies in LLM serving: repeated computation of identical or highly similar prompt prefixes across multiple inference requests. LMCache focuses exclusively on inference acceleration and does not participate in model training or fine-tuning. During transformer-based LLM inference, the attention mechanism generates KV cache tensors for each token in the input sequence. These KV caches represent intermediate computation results that are reused across decoding steps. However, in standard inference pipelines, KV caches are created per request and discarded after completion, even when future requests contain identical or overlapping prompt content. This leads to redundant computation, increased latency, and unnecessary GPU memory usage. LMCache addresses this problem by introducing a reusable KV cache layer that persists KV cache segments across requests. When a new request arrives, LMCache checks whether parts of the input prompt match previously processed token sequences. If a match is found, the corresponding KV cache can be reused directly, allowing the model to skip recomputation for those tokens. This significantly reduces prefill time, especially for long prompts and knowledge-heavy inputs. The core abstraction in LMCache is the segmentation of prompts into reusable cache units. Instead of treating each request as an isolated sequence, LMCache organizes KV caches by token spans or chunks. These chunks can be indexed, stored, and retrieved efficiently. Chunk-based caching allows LMCache to balance reuse granularity and storage overhead, making it suitable for both short and long contexts. LMCache supports multiple storage backends. KV caches can be stored in GPU memory for low-latency reuse, in CPU memory for moderate capacity and flexibility, or in external systems such as Redis or disk-based storage for large-scale and persistent caching. This tiered design enables practical deployment across a wide range of hardware environments and workload patterns. In a typical deployment, LMCache operates alongside an LLM inference engine rather than replacing it. When integrated with inference engines such as vLLM, LMCache intercepts the prefill phase of inference. For tokens whose KV cache already exists, the inference engine directly loads cached KV states instead of recomputing them. For uncached tokens, normal computation proceeds and the resulting KV cache is stored for future reuse. LMCache is particularly effective in scenarios where many requests share common prompt prefixes. Examples include retrieval-augmented generation systems with fixed system prompts, instruction-tuned chat services with standardized templates, and knowledge injection workflows where the same background knowledge is repeatedly supplied to the model. In these cases, LMCache can substantially reduce both latency and GPU compute cost. From a system perspective, LMCache improves inference efficiency without modifying the underlying model architecture. It operates entirely at the execution and memory-management level. This makes it model-agnostic and compatible with a wide range of transformer-based LLMs. Compared to traditional caching mechanisms such as response caching, LMCache works at a much finer granularity. Instead of caching final model outputs, it caches internal attention states. This allows partial reuse even when user queries differ in later tokens, as long as they share a common prefix. As a result, LMCache provides higher cache hit rates and better scalability for interactive and dynamic workloads. It is important to clarify what LMCache is not. LMCache is not a vector database, does not perform semantic retrieval, and does not replace retrieval-augmented generation pipelines. It does not store documents or embeddings. Its sole purpose is to accelerate LLM inference by eliminating redundant computation through KV cache reuse. LMCache is well suited for large-scale LLM serving systems that prioritize throughput, latency, and cost efficiency. It is commonly used in conjunction with high-performance inference engines and scheduling systems, forming a key component in modern LLM serving stacks. In summary, LMCache is an inference-time KV cache reuse system for large language models that reduces redundant computation by persisting and reusing attention KV states across requests, thereby improving latency, throughput, and GPU resource efficiency in real-world LLM deployments.
